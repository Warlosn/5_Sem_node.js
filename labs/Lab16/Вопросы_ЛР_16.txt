1.  GraphQL - формальный язык запросов, позволяющий делать агрегированные запросы.

2.  Схема GraphQL - это структура, составяленная с использованием языка Schema Definition Language (SDL).

3.  Аббревиатура SDL - Schema Definition Language

4.  resolver - инструкции по выполнению операций GraphQL(НАБОР ФУНКЦИЙ, ОБРАБОТЧИК ЗАПРОСОВ)

5.  query - стуктуры, содержащие в себе поля, к которым можно обратиться, а также значения, которые сервер должен вернуть.

6.  mutation - способ модифицировать данные на сервере и получить обработанную информацию. Аналогия концепции CUD (Create, Update, Delete)

7.  subscription - способ поддержки постоянной связи между клиентами и сервером. При обновлении данных сервер выполняет определённый в подписке запрос GraphQL и рассылает обновлённые данные клиентам.

8.  context - значение, предоставляемое каждому resolver'у и содержащее такую контекстуальную информацию, как текущий активной пользователь или доступ к БД.

9.  Модуль graphql - (лекция, большая схема).

10. Описание компонентов:
        * subscription (выше)
        * interface - абстрактный тип, который включает в себя определённый набор полей, которые тип должен подключать в себя для реализации интерфейса.
        * enum - особый вид скаляра, который ограничен определённым набором допустимых значений.
        * fragment - позволяет создавать наборы полей а затем включать их запросы там, где это необходимо.
        * union - тип объединения похож на интерфейсы, но они не задают никаких общих полей между типами.
